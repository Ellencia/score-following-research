# 솔로 개발자 로드맵 — 직장 다니며 Score Following 페이지 터너 만들기

## 0. 한 줄 전략과 성공의 정의

**전략**: 피아노 SOTA를 추격하지 않는다. 분야 빈칸(첼로·일렉기타 + 실사용자 + 끝까지 작동하는 제품)에 "쓸 만한 페이지 터너"를 박고, 그 부산물로 LBD급 기여(데이터셋·user study)를 뽑는다. **모델은 빌리고, 데이터는 본인 활에서 나오는 것만 영구 자산으로 남긴다.**

**성공의 정의(2단 게이트)**:
- **최소 ship선**: 곡 끝까지 추적 *시각화* + 수동(발페달/음성) 전환으로 한 곡을 끊김 없이 마침. 비평2 지적대로 chroma가 plateau여도 도달 가능.
- **풀 MVP선**: 자동 페이지 전환 + 자동 재동기화까지. 측정은 **추적 정확도가 아니라 사용성**으로 — 핵심 지표는 `곡당 사용자 보정 횟수`와 `곡 끝까지 완주율`. "성공" 조작적 정의 = 박자위치 오차 95퍼센타일 ≤ 임계 AND 페이지 전환 누락 0 AND 개입 ≤1회를 한 trial 성공으로, 5회 중 4회.

평가는 **동결된 held-out 곡 5~8개**로만, GT는 추적기 출력이 아닌 독립 소스(수동 탭 정렬 또는 메트로놈 동기 녹음의 박자그리드)로 생성한다(순환논리 회피). 결과는 로그파일 자동 기록.

## 1. 시간선

각 단계는 **한 주말(4~6h 블록)에 끝나고 그 자체로 보여줄 클립이 남는** 슬라이스로 절단한다. 모든 환산은 8h/주 하한 + 30% 버퍼 기준.

### [이번 주말] 생존 스파이크 — 두 게이트
- **게이트0a (라이브러리 sanity)**: `pip install matchmaker`(분석10) → example notebook으로 합성 piano 정렬. ASAP ±5% 확인. *이건 chroma 검증이 아니라 설치 검증일 뿐임을 명심.*
- **게이트0b (가장 위험한 가정 실측, critical)**: 본인 첼로 무반주 16마디 + 일렉기타 clean DI(옵션29) 16마디를 직접 녹음(연주자 해자를 첫 주에 가동). chroma OLTW의 onset 정렬오차를 실측, 합격선 ≤300~500ms 분위. **0b 실패 시 즉시 분기: chroma→CQT/NSGT(Lee Flippy 노선, 코드공개)로 Matchmaker Processor 특징을 한 줄 교체.**
- **동시 개시**: 첼로 {audio.wav + 본인입력 MusicXML + 메타} 페어 적재 시작(follower 0이어도 영구 자산).

### [1단계 0~1개월] 직선 곡 한 페이지 추적
- 일렉기타 단음 라인 + 단일 HTML 빨간 커서가 1페이지를 끝까지 따라감(페이지 전환 없음).
- 일렉기타는 **단음/코드 스트럼 분리 측정**으로 polyphony 모호성을 함수로 본다(비평2 minor).
- **게이트**: 직선 곡 끝까지 커서 유지. **kill 예산**: 3주 통과 못하면 1주만 더, 그래도 실패면 환경/특징 가정 재설계. **안 함**: 페이지 전환, OMR, JLTR, AMT.

### [2단계 1~3개월] 2페이지 + 수동 안전망 먼저
순서를 **수동 fallback 우선**으로 뒤집는다(자동 신뢰 전에 안전망 확보):
1. 2페이지 + 마지막 마디 fade-in 자동 전환 1회.
2. 발페달/Web Speech 'next'(옵션34/35) **수동 전환을 1차 시스템으로**(비평2: plateau 회피 아닌 측정).
3. 탭 재동기화(PIANO PRECISION식, 옵션9) = OLTW window 재초기화.
4. **JLTR 반복 라벨링(옵션8)은 가장 마지막** — 반복 없는 직선 2페이지로 1~3을 먼저 ship하면 늦어져도 80%는 작동.
- **게이트**: 최소 ship선 통과(수동 전환 + 추적 시각화로 완주). **kill**: 4주 후 미달이면 곡 1단계 하향 1회 허용 → 그래도 미달이면 첼로로 전환(monophonic이 chroma 우호적). **누적 3개월·어떤 악기로도 5중3 미달이면 제품 노선 중단, 데이터/user study로 피벗.**
- **안 함**: Causal-AMT fine-tune, distortion, 모바일, 카메라/IMU.

### [3단계 3~6개월] 풀 MVP + 첼로 follower 확장
- 자동 전환·자동 재동기화 안정화 → 풀 MVP선.
- 첼로: PDMX 첼로작품+FluidSynth 사전검증(옵션5) 후 본인 연주. **비브라토를 kill조건이 아니라 핵심 측정변수로** 승격(실제 첼로 비브라토는 ±50~100cents·6~8Hz로 거의 항상 임계 위 = 첼로 디폴트). 완화책: CQT+median smoothing / 비브라토 대역 저역통과. 시연곡 재정의 = "비브라토 약함"이 아니라 **정상 비브라토 + 느린 템포 + onset 명확(피치카토 섞임)** — 데이터 대표성 보존.
- **게이트**: 실사용 게이트(아래 3절). **안 함**: GOAT fine-tune, MERT, federated.

### [4단계 6~12개월] LBD 출구 (둘 병렬, GPU 0)
- **user study(옵션63)**: 최소 ship 단계 산출물로도 N=5 파일럿 가능. 심의 경로 선확정(2.5절).
- **첼로 paired 데이터셋**: 1년 100h↑이면 분야 최초. GT 품질 비종속 설계 — 일렉기타는 옵션30 MIDI pickup의 per-note MIDI를 하드웨어 GT로(chroma 정렬과 무관), 첼로는 메트로놈 동반 버전 동시수집 + Zeitler-Müller 2025 Reformulated Soft-DTW를 약라벨 전제로 채택.
- **kill**: 둘은 **MVP 합격 비종속** — 제품이 막혀도 살아남게 분리. negative result("chroma가 비-피아노 어디서 깨지나")도 첫 LBD 자산.

### [12개월+] 옵션 트랙 (필수 아님)
- GPU 셋업이 풀렸을 때만: 옵션14 MERT×SF 첫 정량비교, 옵션4 GOAT fine-tune, 옵션27 Bow IMU.
- 개인화 서사는 **federated가 아니라 옵션20 few-shot test-time adaptation**(1인·소량 데이터에 맞음). federated는 비전 섹션으로만.

## 2. 세 트랙의 교차 (병렬 환상 제거)

8~12h/주엔 **능동 트랙 1개만** 살아있다. 직렬 백로그를 강제한다:

```
데이터 적재(항상 켜둠·0공수, 주말0부터)
   └─ 제품: 1단계 → 2단계 → 3단계   (active ≤1, queued ≤1)
            └─ 실사용 게이트 통과 후에야 →
   └─ 연구: user study 파일럿 ∥ 데이터셋 공개 (GPU 0, LBD 2건)
            └─ 그 다음에야 → MERT/GOAT (GPU 트랙, 옵션)
```
- WIP 하드상한: active 1 + queued 1. 옵션 카탈로그(70개)는 백로그로 격리, 승격 시 반드시 하나를 done/killed로 비운다.
- **데이터 해자는 제품에 종속 안 됨** — 첼로 페어 적재는 주말0부터(18개월 공백 선점).

## 3. 기술 리스크와 대비책 (정직하게)

- **chroma 비-피아노 미검증(최대 리스크)**: 13편 어디에도 chroma OLTW 비-피아노 정량 없음. 분석11이 피아노에서도 audio OLTW 73% plateau를 입증 → 첼로 80% 사전확률 낮음. → 게이트0b 실측 + CQT/NSGT 즉시 분기 + 사용성 지표로 재정의.
- **sm_120 마찰**: RTX 5080 PyTorch 2.7+/cu128 + CTranslate2 float16 강제. → GPU 셋업을 **독립 선행 spike(2~4주, 산출물=텐서 1개 float16 GPU 통과)**로 분리, fine-tune 일정과 절연. 실패 시 클라우드 GPU.
- **AMT fine-tune 비용**: "48~72h(4090 기준)"는 순수 학습시간일 뿐. 환경+탭전처리+코드정합+augmentation 포함 part-time **6~10주**. → 1차 우회 = 옵션30 MIDI pickup(~$200, ~10ms, distortion 무관, GPU 0)으로 AMT 단계 자체를 하드웨어로 산다. fine-tune은 그게 부족할 때만.
- **첼로/distortion 한계**: 위 비브라토 완화책. 일렉기타 distortion·bend/slide 상시 → Guitar Pro→MusicXML 변환손실률을 사전 1회 정량화(score-side 천장 인지), 시연곡 체크리스트에 bend/slide 빈도 명시.

## 4. 현실 절차 체크리스트 (첫 데이터 전 1회 동결)

- **동의 2층**: ①본인 연주(셀프동의, 공개 시 audio가 본질적으로 식별가능 = 본인 식별 감수 명시) ②미래 제3자(연구공개·재배포·철회권·보관기간). 메타에 `consent_scope`·`identity_link` 처음부터 박기.
- **익명화 정의**: 제거={파일명·일시·위치·장비시리얼}, 보존={파형·정렬라벨}. audio 자체 식별가능성을 README에 disclosure.
- **라이선스 경계**: 공개셋 = 본인 100% 권리(자가연주+자가입력 MusicXML+자가라벨)만. GOAT(CC-BY 4.0 attribution)·PDMX·Klangio·FluidSynth 사운드폰트 파생물은 **내부 학습 전용·공개셋 제외**. 저작권 악보 MusicXML은 곡ID 참조만, audio+라벨만 배포.
- **IRB**: user study는 인간대상연구. 외부/상업 IRB 또는 협력대학 공동저자 경로, 학회별 LBD human-subjects 정책 사전확인. **심의 리드타임 4~8주**를 비용에 명시. 불가 시 N=1 autoethnography+정량로그로 다운그레이드.
- **데이터 거버넌스**: 로컬 저장·서버 전송 없음을 MVP 기본값으로 못박음. 3-2-1 백업(audio 원본 immutable), 라벨에 `label_version`+`generator_model_version`, DVC/git-LFS로 약→강 라벨 이력. 첫 녹음 전 세팅.

## 5. part-time 지속가능성

- **주말 단위 절단**: 모든 슬라이스는 한 블록에 완결 + **15~30초 시연 클립 1개 녹화 의무**(진척의 가시적 증거이자 LBD 데모·README·블로그 재료 = 버려지는 시간 0).
- **시간 배분**: 평일 1h = 라벨링/문서/거버넌스(짧은 슬롯), 주말 블록 = 녹음/통합(통블록). 스위칭 비용↓.
- **멈출 지점 대비**: 모든 게이트에 시도예산+탈출조건(위). "강등" 대신 "범위 축소 후 재-ship". kill조건(비브라토·distortion)은 실패가 아니라 **가치 큰 hard-case 태깅** = 안 되는 경우도 데이터 자산.
- **전체 kill**: 누적 3개월·어떤 악기로도 미달이면 제품 중단, 데이터+user study로 피벗(매몰비용 차단).
