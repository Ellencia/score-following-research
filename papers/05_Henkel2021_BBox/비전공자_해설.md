# Multi-modal Conditional Bounding Box Regression — 비전공자 해설

## 이 논문이 풀려는 문제는 무엇인가

피아니스트가 무대에서 악보를 보며 연주를 하고 있다고 상상해 보자. 악보 옆에는 페이지를 넘겨 주는 사람이 앉아 있다. 이 사람의 일은 단순한 듯 보여도 의외로 어렵다. 음악이 흐를 때마다 지금 연주자가 악보의 어디쯤을 지나고 있는지를 한 박자도 놓치지 않고 따라가야 한다. 너무 빨리 페이지를 넘기면 연주자가 다음 줄을 못 보게 되고, 너무 늦으면 손이 멈춰 버린다. 이 "지금 어디를 연주하고 있나"를 컴퓨터에게 시키는 일이 바로 score following, 한국어로 직역하면 악보 따라가기다. 입력은 두 가지다. 한쪽에는 마이크가 받는 오디오가 있고, 다른 쪽에는 사진처럼 찍힌 악보 이미지가 있다. 출력은 단 하나다. 지금 연주가 닿아 있는 악보 위의 위치.

기존 연구는 이 위치를 표현하는 방법을 두 가지로 시도해 왔다. 첫 번째는 "악보의 모든 픽셀 하나하나에 대해 지금 여기에 연주가 닿아 있을 확률을 계산하라"는 방식이다. 결과는 흐릿한 안개 같은 분포로 나오는데, 보기에도 직관적이지 않고 노트 한 음과 한 픽셀 사이에 명확한 대응이 없다는 약점이 있다. 두 번째는 "악보를 좁고 긴 띠로 펼쳐 놓고 일정한 칸으로 잘라, 어느 칸에 해당하는지 분류하라"는 방식이다. 이건 악보를 미리 잘라 펼치는 전처리가 필요하고, 칸을 너무 크게 잡으면 정밀도가 떨어진다.

이 논문이 던지는 질문은 단순하다. "왜 점이나 안개로 표시하지? 그냥 직사각형으로 둘러싸면 안 되나?"

## 핵심 아이디어를 한 그림으로

스마트폰의 카메라 앱에 얼굴을 비추면 노란 사각형이 얼굴 둘레에 나타난다. 사진 속 자동차나 강아지를 인식하는 인공지능도 같은 방식으로 작동한다. 그 직사각형을 학계에서는 bounding box, 우리말로 경계 상자라고 부른다. 이 논문은 score following에도 똑같은 아이디어를 가져온다. 매 순간 들리는 음악에 대해 "지금 막 친 음표(또는 화음)는 악보의 이 직사각형 영역에 있다"고 컴퓨터가 작은 박스를 그리도록 하는 것이다.

```mermaid
flowchart LR
    오디오[현재 들리는 음악] --> 인코더[음악을 숫자로 요약]
    악보[악보 이미지 한 페이지] --> 비전[그림을 보는 신경망]
    인코더 -. 조건 .-> 비전
    비전 --> 박스[악보 위의 직사각형 + 신뢰 점수]
```

비유를 하나 더 들어 보자. 사진 검색에서 "빨간 모자를 쓴 아이를 찾아 줘"라고 말하면 인공지능이 사진을 훑어 보고 빨간 모자 아이만 박스로 표시해 준다. 이때 "빨간 모자를 쓴 아이"는 검색 조건(query)이다. 이 논문에서는 글로 된 검색어 대신 오디오가 검색 조건이 된다. 즉 "지금 들리는 이 짧은 음악 조각에 해당하는 부분을 악보에서 박스로 찾아 줘"라고 컴퓨터에게 부탁하는 셈이다. 영어로는 conditional object detection, 직역하면 조건부 객체 검출이다.

오디오가 조건으로 작동하는 방식은 FiLM이라 불리는 기제로 구현된다. FiLM은 비유하자면 안경의 색 필터 같다. 같은 풍경을 봐도 빨간 필터를 끼면 빨간 부분이 도드라지고, 파란 필터를 끼면 파란 부분이 도드라진다. 마찬가지로 오디오에서 추출한 짧은 요약 벡터가 시각 신경망의 중간 단계마다 "이번엔 이런 음역대의 음표를 강조해서 봐"라고 필터를 갈아 끼워 주는 역할을 한다.

## 알아야 할 핵심 용어

| 용어 | 영문 | 직관적 설명 |
|---|---|---|
| 경계 상자 | Bounding Box | 사진이나 그림 속 어떤 대상을 둘러싸는 직사각형. 좌표 네 개(중심 x, y와 폭, 높이)로 표현된다. |
| 객체 검출 | Object Detection | 이미지에서 "어떤 것"이 "어디"에 있는지 동시에 찾아내는 분야. 얼굴 인식의 노란 박스가 대표 사례다. |
| 분할 | Segmentation | 박스가 아니라 픽셀 단위로 영역을 칠하듯 표시하는 방법. 더 정밀하지만 출력이 무겁고 의미 부여가 어렵다. |
| 회귀 | Regression | 정답이 숫자(예: 좌표값)인 예측. 분류(고양이/개)는 카테고리, 회귀는 연속적인 값이다. 박스 위치 예측은 회귀에 해당한다. |
| IoU | Intersection over Union | 예측한 박스와 정답 박스가 겹치는 면적을 두 박스 합집합 면적으로 나눈 비율. 0이면 안 겹친 것, 1이면 완벽히 겹친 것. |
| 조건부 입력 | Conditional Input | 같은 신경망이라도 외부에서 들어오는 추가 정보(여기서는 오디오 요약)에 따라 출력이 달라지게 하는 입력. |
| FiLM | Feature-wise Linear Modulation | 신경망의 중간 특징 지도(feature map)를 외부 조건에 따라 곱하고 더해(scale & shift) 휘는 기법. 비유: 색 필터 안경. |
| 앵커 | Anchor | 박스를 예측할 때 참고하는 기준 사각형. 모델은 앵커로부터 얼마나 떨어지고 얼마나 커졌는지(offset)만 예측하면 된다. |
| 신뢰 점수 | Confidence / Objectness | 예측한 박스가 실제로 정답과 잘 맞을지에 대한 자체 점수. 클수록 자신있다는 뜻. |
| YOLO | You Only Look Once | 이미지 한 번에 박스와 신뢰도를 동시에 뽑아내는 빠른 객체 검출 모델 계열. 실시간 응용에 적합. |
| MSMD | Multi-modal Sheet Music Dataset | 악보 이미지와 그에 대응하는 합성 피아노 오디오·MIDI를 정렬해 놓은 공개 데이터셋. |
| 임펄스 응답 | Impulse Response | 어떤 공간(콘서트홀, 사무실 등)이나 마이크가 짧은 자극에 어떻게 반응하는지를 기록한 짧은 소리. 이를 곱하면 그 공간/마이크에서 녹음한 듯한 효과를 모사할 수 있다. |
| 데이터 증강 | Data Augmentation | 학습 데이터를 일부러 변형해 불려 모델이 다양한 상황에 강해지도록 만드는 기법. |
| 합성-실제 갭 | Synthetic-to-Real Gap | 컴퓨터로 만든 깔끔한 학습 데이터와 실제 환경에서 들어오는 지저분한 데이터 사이의 성능 차이. |

## 이 논문의 새로운 점

가장 큰 변화는 출력의 형태다. 이전의 대표 방식인 CUNet은 악보 페이지의 모든 픽셀에 "여기일 확률"을 칠해 주는 그림 분할 방식이었다. 이 논문은 그 대신 단 하나의 직사각형을 그리도록 모델을 다시 짠다. 점 하나가 아니라 박스라는 점도 중요하다. 점은 정확한 좌표 한 곳을 가리키지만, 음표나 화음은 사실 작은 영역을 차지한다. 박스를 쓰면 폭과 높이를 통해 "이 화음 정도의 영역"을 자연스럽게 표현할 수 있고, 박스끼리는 서로 명확히 구분되므로 모델이 여러 후보 위치를 두고 헷갈릴 때 어떤 것이 더 그럴듯한지 점수를 매겨 비교하기도 쉬워진다.

직관적인 비교를 하자면 이렇다. 점 회귀는 지도 위에 핀 하나를 콕 찍는 것이다. 분할은 지도 위에 안개로 어디쯤이라고 흐릿하게 칠하는 것이다. 박스 회귀는 지도 위에 사각형 영역을 그리는 것이다. 지도에서 길찾기를 한다면 핀이 좋겠지만, "어떤 동네인지" 같은 정보를 함께 보고 싶다면 사각형이 더 유용하다. 악보를 따라갈 때는 "어떤 음표(또는 화음)인지"를 함께 보여주는 사각형 쪽이 의미적으로 더 적합하다는 것이 이 논문의 입장이다.

두 번째 새로운 점은 실제 녹음에 대응하는 방식이다. 이런 종류의 모델은 보통 깔끔한 합성 음원으로 학습한다. 진짜 마이크 녹음으로는 정렬 라벨을 만들기가 너무 비싸기 때문이다. 그러다 보니 모델이 합성 음원에는 강하지만 실제 마이크 녹음으로 가면 성능이 뚝 떨어지는 고질적인 문제가 있었다. 이 논문은 학습 도중에 임펄스 응답 500여 개를 무작위로 오디오에 곱해 주는 단순한 트릭을 도입한다. 마치 아이가 같은 노래를 거실에서, 욕실에서, 강당에서 번갈아 들으면서 자라면 어디서 들어도 같은 노래임을 알아차리듯이, 모델은 다양한 공간 음향을 거치며 일반화 능력을 키운다. 실험 결과 사무실에서 마이크로 녹음한 피아노에 대해 0.05초 정확도가 약 31%에서 약 56%로 거의 두 배가 된다. 추가 데이터를 모은 것이 아니라 학습 시점에 약간의 잔향만 섞었을 뿐인데도 그렇다.

세 번째 새로운 점은 모델 골격이다. 이전 분할 방식은 U-Net을, 이 논문은 YOLO를 가져온다. YOLO는 자율주행이나 영상 감시에서 실시간 객체 검출의 대명사다. 빠르고 가볍기 때문에 약 20분의 1초 단위로 들어오는 오디오 프레임을 실시간으로 처리해야 하는 score following과 궁합이 좋다. 또 YOLO에는 신뢰 점수라는 출력이 자연스럽게 들어 있어서, 박스가 얼마나 믿을 만한지를 모델 스스로 평가하게 만들 수 있다.

## 한계와 의의

물론 한계도 있다. 첫째, 평가가 모두 피아노 음악에 한정된다. 솔로 바이올린, 챔버 트리오, 오케스트라처럼 음색이 다양해지고 동시에 여러 줄의 악보를 따라가야 하는 경우에는 어떻게 작동할지 이 논문만으로는 알 수 없다. 둘째, 박스의 폭은 학습 단계에서 30 픽셀로 임의로 정해진 상수다. 다시 말해 이 논문에서 박스 폭은 의미적으로 큰 정보를 담지 않으며, 정밀한 평가는 결국 시간축 기준의 onset 정확도로 환원된다. 박스라는 형태가 가져다주는 진짜 이득은 출력의 명료함과 신뢰 점수, 다중 후보 비교의 가능성이지 박스의 폭 자체는 아니다. 셋째, 실제 녹음에서 5초 이내에 따라잡는 비율을 보면 전통적인 OMR 기반 방법이 여전히 경쟁력을 유지한다. IR 증강은 큰 도움이 되지만 합성-실제 갭을 완전히 메우지는 못했다.

그럼에도 불구하고 의의는 크다. 첫째, score following의 출력 표상에 대해 새로운 길을 열었다. 점도 분할도 아닌 박스라는 선택이 자연스럽고 확장 가능하다는 것을 보여 준다. 저자들이 후속 연구로 제안하는 마디 단위, 시스템(악보 한 줄) 단위 박스로의 확장은 이 표상이 갖는 유연성을 잘 보여 준다. 둘째, 합성-실제 갭이라는 오래된 문제에 대해 추가 데이터 없이 작동하는 단순하고 강력한 해법을 제시한다. 이는 같은 종류의 도메인 갭 문제를 가진 다른 멀티모달 작업(음성-이미지, 음성-텍스트 등)에도 시사하는 바가 크다. 셋째, 자동 페이지 넘김, 콘서트홀의 실시간 자막, 인공지능 반주자처럼 이미 상상되어 온 응용에 한 걸음 더 가까이 다가갔다. 박스는 화면 위에 직접 그려 보여 주기 좋은 형태이므로 사용자 경험 측면에서도 직관적인 인터페이스를 만들기 쉽다.

요약하자면 이 논문은 "악보를 따라간다"는 오래된 문제를 "악보 위에 사각형을 그린다"는 친숙한 문제로 옮겨 놓는다. 이 단순한 시각의 전환이 정확도를 높이고, 실세계 일반화의 길을 열고, 후속 확장의 여지까지 만들어 낸다는 점에서, 작은 논문이지만 분야의 흐름을 바꾸는 한 걸음이라 부를 수 있다.
