# Exploring System Adaptations for Minimum Latency Real-Time Piano Transcription — 비전공자 해설

## 이 논문이 풀려는 문제는 무엇인가

먼저 한 가지 구분이 필요하다. 컴퓨터 음악 분야에서 "online"과 "real-time"은 같은 말이 아니다. **Online**은 입력을 청크 단위로 받아 점진적으로 출력한다는 뜻이지만 지연이 100-300 ms 수준이어도 online이라 부른다. 반면 **real-time**은 사람이 음악적으로 자연스럽게 인터랙션할 수 있는 지연 — 보통 30 ms 이하, 가급적 10 ms 이하 — 을 요구한다. 디지털 피아노에 키를 눌렀을 때 100 ms 후에 소리가 나면 누구나 어색함을 느끼고, 화상 합주에서도 30 ms를 넘으면 박자가 흔들린다.

본 논문이 다루는 자동 음악 전사(AMT) — 마이크로 들어오는 피아노 소리를 실시간으로 음표 시퀀스로 바꾸는 작업 — 는 같은 부정확한 명명을 안고 있다. 현재 SOTA "online" AMT의 지연은 128-320 ms로, 자동 반주·디지털 악기 같은 응용에는 너무 크다. 본 논문은 이 격차를 정조준한다. 출발점은 Mobile-AMT(Kusaka-Maezawa, EUSIPCO 2024)라는 SOTA online 모델로 보고된 지연 174 ms인데, 저자들은 그 안에 숨은 두 가지 함정을 지적한다 — Squeeze-Excitation 층은 청크 전체를 보므로 사실 10 초의 lookahead가 있고, 후처리는 onset 주변 삼각형의 max를 찾기 위해 미래 프레임을 봐야 한다.

목표는 분명하다. **30 ms(또는 가급적 10 ms) 지연에서 동작하는 진짜 real-time AMT를 만들어, 그 baseline을 오픈 소스로 풀어 분야의 후속 연구를 유도한다.** 결과는 솔직하다 — 30 ms 지연에서는 비-causal 베이스라인과 비슷한 정확도가 가능하나, 10 ms 지연을 강제하면 정확도가 크게 떨어진다. 그러나 적어도 분야는 이제 어디까지 좁혔고 어디가 어려운지를 정량적으로 알게 되었다.

## 한 줄 비유로 본 이 논문의 아이디어

**"미래를 한 번도 보지 않고 — 정말 한 번도 — 음표를 잡아낸다."**

비유하자면, 야구 경기를 중계하는 캐스터를 생각해 보자. 보통의 캐스터는 이미 끝난 1초를 살짝 되돌아 보고 더 정확한 멘트를 한다. 하지만 본 논문이 만들고자 하는 캐스터는, 절대로 과거 1초도 되돌아 보지 못하고 매 순간 들어오는 소리만으로 즉시 멘트를 해야 한다. 이렇게 하면 멘트가 부정확해지지만 청취자가 느끼는 지연이 거의 없다. 본 논문은 이 "절대 미래를 안 본다"는 strict causality 제약을 모델 모든 곳에 적용하고, 그로 인한 정확도 손실을 다른 부분에서 어떻게 회복할 수 있는지를 체계적으로 ablation한다.

## 핵심 아이디어를 한 그림으로

```mermaid
flowchart LR
    Mic[마이크 입력] --> STFT[STFT<br/>비대칭 윈도우<br/>10-30 ms shift]
    STFT --> Mel[Mel 스펙트로그램]
    Mel --> Conv[Causal Conv<br/>SE 층 제거]
    Conv --> Stack[공유 Acoustic Stack<br/>onset/frame/velocity]
    Stack --> Post[Causal Postprocessing<br/>현재>임계 + 직전<임계]
    Post --> Notes[음표 onset 시퀀스]
```

전체 파이프라인의 모든 단계가 strict causal — 미래 정보 미사용 — 을 보장하도록 다시 짠다. (1) STFT는 윈도우를 미래 쪽으로 작게만 두어 lookahead를 10-30 ms로 줄이고, 보통의 Hann 윈도우가 경계 정보를 너무 강하게 감쇠시키는 점을 비대칭 함수로 보완한다. (2) 컨볼루션은 receptive field를 과거 8 + 현재 1 프레임으로 비대칭 변경. SE 층은 통째로 제거. (3) acoustic stack 셋(onset/frame/velocity)은 모든 conv 층을 공유해 모델 크기를 절반(320 → 160 GFLOPs)으로 줄임. 동시에 velocity 조건이 onset 예측을 방해한다는 의외의 발견을 이용해 그 의존성을 끊음. (4) 학습 target도 onset 주변 삼각형이 아닌 그 정확한 한 프레임만 1로 두는 binary point target으로 바꿔, 후처리도 strict causal로 단순화한다.

## 알아야 할 핵심 용어

| 용어 | 영문 | 직관적 설명 |
|------|------|----------|
| AMT | Automatic Music Transcription | 마이크 입력을 매 순간 어떤 음이 시작/지속/끝나는지의 음표 시퀀스로 변환 |
| Online vs Real-time | Online vs real-time | online은 청크로 받아 점진 출력(100-300 ms 가능), real-time은 30 ms 이하 지연 — 본 논문은 이 둘을 분명히 구분 |
| Causal | Causal | 미래 정보를 일절 사용하지 않는 처리. 음악 인터랙션에는 필수 |
| 지연 (Latency) | Latency | 입력이 들어가서 출력이 나오기까지의 시간. 본 논문은 algorithmic latency(STFT shift + causal conv 등)를 측정 |
| STFT | Short-Time Fourier Transform | 오디오를 작은 윈도우로 잘라 각 윈도우의 주파수 성분을 보는 것 |
| Mel 스펙트로그램 | Mel spectrogram | STFT를 사람 청각에 맞춘 mel 척도로 변환한 것. AMT의 표준 입력 |
| Hann 윈도우 | Hann window | STFT의 표준 윈도우 함수. 양쪽 경계를 부드럽게 감쇠시킴 |
| 비대칭 윈도우 | Asymmetric window | 양쪽 길이가 다른 윈도우. lookahead를 줄이면서 최근 샘플 정보를 유지 |
| Squeeze-Excitation (SE) | Squeeze-Excitation layer | MobileNet V3에 들어 있는 채널 재가중 모듈. spatial 평균을 보므로 본질적으로 비-causal |
| Triangular target vs Binary point target | Regression target shapes | 음표 onset 주변에 삼각형으로 펼친 target은 sub-frame 정확도가 가능하나 lookahead 필요. 한 프레임만 1로 두면 strict causal 가능 |
| Shift-tolerant loss | Shift-tolerant loss | ±1 프레임 오차를 허용하는 loss. 정확도가 좋지만 strict causal에서는 systematic delay를 만들 수 있음 |
| BCE | Binary Cross-Entropy | 이진 분류 표준 loss. 클래스 불균형을 다루기 위해 양성에 가중치를 주는 방식 |
| Acoustic stack | Acoustic stack | onset/frame/velocity 같은 target별 학습되는 conv 블록 묶음. 본 논문은 이를 target 간 공유 |
| Velocity conditioning | Velocity conditioning | velocity 예측 결과를 onset 예측에 조건으로 넣는 방식. 본 논문이 발견한 — 이걸 빼면 onset이 더 정확함 |
| MAESTRO | Dataset | 200 시간 분량의 piano competition 녹음 + 정밀 MIDI 라벨 데이터셋(Hawthorne 2019) |
| receptive field | Receptive field | 신경망 한 출력이 보는 입력 시간 범위. 본 논문은 이를 과거 8 + 현재 1로 비대칭 설정 |
| F1 score | F1 | precision과 recall의 조화 평균. AMT의 표준 평가 지표 |
| onset tolerance | Onset tolerance | 평가에서 "맞췄다"고 인정해 주는 시간 허용 범위. 보통 ±50 ms이지만 real-time 평가에는 ±10/20/30 ms 사용 |
| Mobile-AMT | Reference model | Kusaka-Maezawa 2024의 SOTA online AMT. 본 논문의 출발점 |

## 이 논문의 새로운 점

크게 세 가지를 들 수 있다.

**첫째, "online ≠ real-time"이라는 정직한 진단.** 분야는 자기 모델이 "online이다"라고 말하는 것을 마치 real-time과 동의어로 사용하는 경향이 있었다. 본 논문은 이 통념을 정조준해, 30 ms는 networked ensemble의 표준이고 10 ms는 디지털 악기의 perception threshold이라는 인접 분야의 정량 근거를 인용해 real-time의 기준을 새로 정의한다. 그리고 출발점인 Mobile-AMT의 보고된 지연 174 ms가 SE 층의 10 초 lookahead 때문에 사실은 더 크다는 점을 지적한다. 이 지점부터 분야의 솔직성이 한 단계 올라간다.

**둘째, 세 갈래의 체계적 ablation.** 학습 target과 후처리, 오디오 전처리, 모델 아키텍처 — 세 단계 각각을 독립 실험한 뒤 마지막에 결합한다. 결과는 분명하다. (i) 학습 단계에서 binary point target + weighted BCE는 triangular regression과 비슷한 정확도를 strict causal 형태로 달성, (ii) 전처리 단계에서 비대칭 윈도우 30 ms shift는 비-causal 베이스라인과 같은 정확도, 10 ms shift는 정확도 큰 폭 손실, (iii) 아키텍처 단계에서 acoustic stack 공유는 모델 크기를 절반으로 줄이면서도 성능 유지, velocity conditioning 제거는 오히려 onset 정확도 향상. 어느 단계가 정확도를 가장 손해 보는가 — 저자들의 결론은 "shifted window + causal convolutions"이며, 이 발견은 후속 연구가 어디를 더 파야 할지를 분명히 한다.

**셋째, 정직한 trade-off 보고와 baseline 공개.** 최종 비교에서 본 논문 Causal-AMT는 strict timing tolerance(10 ms)에서는 Mobile-AMT를 앞서지만, 보통 metric(30 ms tolerance)에서는 비-causal Mobile-AMT가 앞선다. 저자들은 이 결과를 숨기지 않고 명시적으로 보고하고, 본 시스템을 baseline으로 공개해 후속 연구의 출발점이 되도록 의도한다. 분야가 후속 연구로 어디를 메워야 하는지를 분명한 baseline으로 제시하는 — 이런 류의 negative-yet-honest 논문이 분야 발전에 큰 기여를 한다.

## 한계와 의의

**한계.** 첫째, 평가가 MAESTRO(깔끔한 Disklavier 녹음)에 한정된다. Mobile-AMT가 강조한 "다양한 acoustic distribution shift" 강건성은 본 논문에서 평가에서 빠져 있다 — 저자도 future work로 명시. 둘째, wall-clock 측정이 없다. 알고리즘적 지연(STFT shift + causal conv 등)은 정밀히 다루지만, 실제 GPU/CPU에서의 inference time + OS overhead가 30 ms 안에 들어가는지는 검증되지 않음. 셋째, 비-피아노로의 확장 부재 — 본 논문은 피아노 한정으로, 바이올린·첼로·기타로의 일반화는 별도 작업이 필요하다(이 프로젝트의 페이지 터너 관점에서 가장 큰 격차).

**의의.** 학술적으로 이 논문은 분야에 정직한 baseline을 제공하는 류의 인프라적 기여다. "online ≠ real-time"이라는 분류를 정량으로 깔끔히 정리하고, 그 격차를 좁히는 첫 시도를 체계적 ablation으로 보여 줬다. 본 프로젝트의 페이지 터너 관점에서는 단독으로는 가치가 작은 편이지만(페이지 터너에 174 ms는 충분히 작음), 이 분석 12번 + 분석 11번(transcribe-then-track) + 분석 10번(Matchmaker) 셋이 결합될 때 가치가 결정적이다. 분석 10번이 백엔드 라이브러리이고, 분석 11번이 그 안에 끼워 넣을 가장 정확한 정렬 모듈이며, 분석 12번이 분석 11번의 174 ms 지연을 30 ms 수준으로 떨어뜨리는 저지연 AMT 모듈이다. 셋이 한 응용 스택을 이룬다는 점에서, 이 세 편의 ISMIR/SMC 2025 동시 발표는 JKU 그룹이 분야의 다음 10년을 동시에 준비한 결과로 읽힌다.
